# 需求文档

## 1. 应用概述

### 1.1 应用名称
城市急救 120 秒

### 1.2 应用描述
公益主题浏览器端轻量3D急救模拟游戏。玩家在城市社区内扮演第一目击者，在120秒内完成急救关键步骤。通过可互动、可复盘、可传播的游戏体验，降低公众接触急救知识的心理门槛。

### 1.3 产品目标
- 在3-5分钟内让玩家体验一次完整的城市急救行动链
- 将急救知识转化为可操作的游戏动作
- 通过倒计时、节奏判定、NPC协作和小红花奖励形成明确游戏反馈
- 控制技术难度，保证浏览器可稳定运行、展示流程清晰、可稳定复现

### 1.4 医学安全声明
游戏内所有急救步骤仅用于公益科普和情景模拟。开始页和结算页需注明「本游戏不能替代红十字会、急救中心或其他专业机构的线下培训」。实际急救应优先拨打当地急救电话，遵循急救调度员、AED设备语音提示和经过认证的急救培训要求。

## 2. 用户与使用场景

### 2.1 目标用户
- **玩家**：通过键盘和鼠标在社区场景中移动、观察突发事件、选择急救动作、分配旁观者任务，并根据系统提示完成关键操作
- **体验观察者**：关注作品完整性、玩法成立性、公益主题嵌入、技术难度与完成度匹配
- **演示讲解者**：负责现场或录屏展示游戏流程，讲解急救链条、产品社会价值和后续扩展空间

### 2.2 核心使用场景
用户打开浏览器进入游戏，选择开始后进入城市社区广场场景，在120秒倒计时内发现突发事件、确认现场安全、呼叫120、指派旁观者取AED、判断患者意识和呼吸、进行胸外按压节奏操作、使用AED设备、等待救护车到达，最终获得小红花评分和急救流程复盘建议。

## 3. 页面结构与功能说明

### 3.1 页面结构

```
城市急救 120 秒
├── 开始页
├── 教学页
├── 游戏页
├── 结算页
└── 开发说明页（可选）
```

### 3.2 开始页

**功能说明**：
- 展示游戏名称「城市急救 120 秒」
- 展示公益主题说明
- 提供开始按钮进入教学页
- 显示医学安全声明：「本游戏不能替代红十字会、急救中心或其他专业机构的线下培训」

### 3.3 教学页

**功能说明**：
- 在30秒内说明基础操作：WASD移动、鼠标视角或固定俯视角、E键交互、数字键1-4选择急救动作、空格键按压节奏
- 说明急救链条核心步骤：确认现场安全 → 呼叫120 → 指派取AED → 判断意识和呼吸 → 胸外按压 → AED操作 → 等待救援
- 提供进入游戏按钮

### 3.4 游戏页

**3.4.1 3D场景**

场景名称：城市社区广场

地图参数：
- 地图大小：28 x 28 Three.js单位
- 玩家初始位置：(0, 1.6, 9) 或俯视角 (0, 18, 14)
- AED站点位置：(-8, 0, -5)
- 患者位置：(3, 0, -3)
- 救护车到达点：(0, 0, -13)
- 可移动边界：X/Z约 [-13, 13]

场景元素：
- 社区广场地面和道路
- AED站点
- 倒地患者
- 旁观者NPC
- 急救车到达点
- 小红花墙
- 路牌/公益海报（展示「人人学急救，急救为人人」等主题语）
- 长椅、便利店等环境装饰

**3.4.2 玩家控制**

- WASD键移动
- 鼠标控制视角或固定俯视角
- E键与场景元素交互
- 数字键1-4选择急救动作
- 空格键进行胸外按压

**3.4.3 HUD布局**

- 左上角：当前事件名称、急救阶段
- 正上方：120秒倒计时
- 右上角：小红花数量、步骤正确率
- 左下角：操作提示
- 右下角：当前可用动作按钮
- 中央：交互准星或目标高亮
- 底部：胸外按压节奏条（按压小游戏时显示）

**3.4.4 急救动作系统**

10个急救动作：

| 动作ID | 动作名称 | 输入方式 | 说明 |
|---|---|---|---|
| check_scene | 确认现场安全 | E键/菜单 | 检查车辆、电线、火源等环境风险 |
| call_120 | 呼叫120 | 数字键1/菜单 | 触发急救车倒计时 |
| assign_aed | 指派取AED | 数字键2/菜单 | 让旁观者去AED站点 |
| check_response | 判断意识 | 数字键3/菜单 | 呼唤、轻拍肩部 |
| check_breath | 判断呼吸 | 数字键4/菜单 | 判断呼吸状态 |
| compression | 胸外按压 | 空格键 | 进入节奏小游戏 |
| power_aed | 打开AED | E键/菜单 | AED到达后可用 |
| attach_pads | 贴AED片 | 点击高亮区域 | 点击两个高亮区域完成贴片 |
| clear_patient | 确认无人接触 | E键/菜单 | AED分析/电击前必须执行 |
| continue_cpr | 继续按压 | 空格键 | 电击后继续按压 |

**3.4.5 按压节奏小游戏**

玩家按空格键进行胸外按压，系统根据相邻两次按压的时间间隔进行判定：

| 判定 | 间隔范围 |
|---|---|
| perfect | 500-600 ms |
| good | 460-680 ms |
| slow | > 680 ms |
| fast | < 460 ms |

显示节奏条，目标区间为100-120次/分钟的节奏感。连续good以上可积累「稳定按压」进度。长时间断按会扣分并触发提示音。

### 3.5 结算页

**功能说明**：
- 展示得分和小红花数量
- 展示正确流程复盘
- 列出错误步骤提示
- 显示医学安全声明
- 提供「再来一次」按钮返回开始页

### 3.6 开发说明页（可选）

**功能说明**：
简述剧情、美术、文案、代码等内容的生成和整理过程。

## 4. 业务规则与逻辑

### 4.1 关卡设计

游戏包含3个急救事件，MVP阶段完整实现关卡1，关卡2和关卡3作为简化教学事件。

**关卡1：广场篮球场成人突然倒地**

事件描述：玩家看到一名NPC在篮球场边突然倒地，系统提示「有人倒地，请靠近查看」。

正确流程（12步）：
1. 靠近患者
2. 确认现场安全
3. 呼叫120
4. 指派旁观者取AED
5. 判断患者意识
6. 判断呼吸状态
7. 开始胸外按压节奏小游戏
8. AED到达后打开AED
9. 按提示贴片
10. 确认无人接触患者
11. 完成模拟电击
12. 继续按压直到救护车到达

失败或扣分行为：
- 未呼叫120就开始按压
- 未确认现场安全
- 忘记指派AED
- 按压节奏过慢或过快
- AED分析或电击时仍接触患者
- 长时间无操作

**关卡2：老人跌倒（简化教学）**

玩家需要靠近老人，选择「不要贸然搬动」「询问情况」「呼叫帮助」。此关不做复杂医学判断，重点表达适老化关怀和安全意识。

**关卡3：高温中暑（简化教学）**

玩家需要将NPC引导到阴凉处、补充水分提示、呼叫帮助。此关主要用于扩展公益主题。

### 4.2 动作状态机

急救流程状态转换：

incident_found（发现事件）→ scene_safe（现场安全）→ emergency_called（已呼救）→ aed_requested（已指派AED）→ response_checked（已判断意识）→ breath_checked（已判断呼吸）→ cpr_active（按压中）→ aed_ready（AED就绪）→ pads_attached（已贴片）→ clear_confirmed（已清场）→ shock_done（已电击）→ cpr_active（继续按压）→ rescued（救援到达）

### 4.3 时间规则

单局时长120秒，建议时间分配：
- 发现事件：0-15秒
- 呼救与分工：15-35秒
- 判断与按压：35-85秒
- AED操作：85-110秒
- 救援到达与结算：110-120秒

### 4.4 评分系统

分数构成：

| 指标 | 权重 | 说明 |
|---|---|---|
| 流程正确率 | 40% | 是否按正确顺序完成关键步骤 |
| 时间效率 | 20% | 呼救、取AED、开始按压是否及时 |
| 按压节奏 | 20% | 节奏小游戏稳定度 |
| 错误控制 | 10% | 错误操作越少越好 |
| 公益协作 | 10% | 是否合理分配旁观者任务 |

小红花换算：

| 结果 | 小红花 |
|---|---|
| S级 | 5朵 |
| A级 | 4朵 |
| B级 | 3朵 |
| C级 | 2朵 |
| 未完成 | 1朵鼓励花 |

小红花不是金钱或真实捐赠凭证。

### 4.5 音效规则

| 音频 | 说明 |
|---|---|
| 背景环境音 | 城市广场、人群低语、远处车流 |
| 倒计时音 | 关键时间节点提示 |
| 按压节拍音 | 100-120 BPM范围内的节拍参考 |
| AED提示音 | 模拟「打开设备、贴片、请勿接触」等提示 |
| 救护车声 | 随救援到达逐渐增强 |
| 小红花音效 | 成功步骤和结算奖励 |

MVP使用Web Audio API生成简洁提示音。

### 4.6 视觉风格规则

- 城市黄昏/夜晚社区，气氛紧张但不恐怖
- 主色：深蓝、急救红、暖黄色、小红花红
- 低多边形风格，基础几何体
- 不使用血腥伤口表现
- 患者状态用轮廓、姿态、呼吸图标和UI提示表达

## 5. 异常与边界情况

| 场景 | 处理方式 |
|---|---|
| 120秒倒计时结束 | 强制进入结算页，显示未完成结果和1朵鼓励花 |
| 玩家移动超出边界 | 限制在X/Z约[-13, 13]范围内 |
| 错误操作顺序 | 扣分并在HUD显示提示，不阻断流程 |
| 长时间无按压操作 | 触发提示音和文字提示 |
| AED分析时接触患者 | 扣分并显示警告 |
| 按压节奏持续过快或过慢 | 降低节奏评分，显示调整提示 |

## 6. 数据模型

### 6.1 GameState

包含字段：
- phase：游戏阶段（start/briefing/playing/result）
- timeLeft：剩余时间（秒）
- currentScenarioId：当前关卡ID
- currentStep：当前急救步骤
- flowers：小红花数量
- score：得分
- mistakes：错误操作列表
- completedSteps：已完成步骤列表
- compressionStats：按压统计数据
- ambulanceEta：救护车预计到达时间
- aedEta：AED预计到达时间

### 6.2 CompressionStats

包含字段：
- totalPresses：总按压次数
- perfect：perfect判定次数
- good：good判定次数
- slow：slow判定次数
- fast：fast判定次数
- longestCombo：最长连击数
- lastPressAt：上次按压时间戳

### 6.3 Scenario

包含字段：
- id：关卡ID
- title：关卡标题
- description：关卡描述
- location：患者位置坐标[x, y, z]
- requiredSteps：必需步骤列表
- educationalNote：教育说明

## 7. 技术栈说明

本项目使用以下技术栈：
- 应用框架：React + TypeScript
- 构建工具：Vite
- 3D渲染：Three.js / React Three Fiber
- 3D辅助：drei
- 状态管理：Zustand或React useReducer
- UI样式：Tailwind CSS
- 图标：lucide-react
- 动画：motion
- 音效：Web Audio API
- 部署：静态Web部署（Frontend Game类型）

推荐目录结构：
```
src/
  App.tsx
  main.tsx
  game/ (state.ts, rules.ts, scoring.ts, scenarios.ts, constants.ts)

  scene/ (CommunityScene.tsx, PlayerController.tsx, NPC.tsx, Patient.tsx, AEDStation.tsx, Ambulance.tsx)
  audio/ (sound.ts)
  styles/ (index.css)
```

## 8. 验收标准

1. 用户打开浏览器访问游戏网址
2. 点击开始按钮进入教学页，阅读操作说明
3. 进入游戏页，在城市社区广场场景中发现倒地患者
4. 使用WASD移动靠近患者，按E键确认现场安全
5. 按数字键1呼叫120，按数字键2指派旁观者取AED
6. 按数字键3判断意识，按数字键4判断呼吸
7. 按空格键进行胸外按压，保持100-120次/分钟节奏
8. AED到达后按E键打开AED，点击高亮区域贴片
9. 按E键确认无人接触患者，完成模拟电击
10. 继续按空格键按压直到救护车到达
11. 进入结算页查看小红花数量、得分和急救流程复盘
12. 点击「再来一次」按钮重新开始游戏
